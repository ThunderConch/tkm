# PvP 서버 Storage 문서

상위 문서: [서버 설계 인덱스](../README.md)

이 섹션은 서버 권한 PvP를 뒷받침하는 **엔티티 구조 / 저장 책임 / DB 스키마**를 정리한다.
즉, “무엇을 어떤 단위로 저장하고, 어떤 제약으로 무결성을 유지할지”를 설명하는 하위 묶음이다.

## 포함 문서

1. [데이터 모델](./data-model.md)
2. [DB 스키마 초안](./database-schema.md)

## 읽는 순서

- 먼저 [데이터 모델](./data-model.md)로 엔티티 관계와 책임 경계를 본다.
- 그 다음 [DB 스키마 초안](./database-schema.md)으로 컬럼 / 제약 / 인덱스 수준까지 내린다.

## 이 묶음이 답하는 질문

- 온라인 파티 스냅샷은 어떤 생명주기를 가지는가?
- 룸, 배틀, 턴, 명령, 이벤트는 어떻게 분리 저장되는가?
- anti-cheat 관점에서 어떤 로그와 무결성 필드가 필요한가?
